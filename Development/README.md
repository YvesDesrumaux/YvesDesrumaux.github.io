# AL/Business Central Development Guidelines

## Workflow Purpose

Development provides fundamental AL language guidance and object development patterns for Microsoft Dynamics 365 Business Central. This workflow covers :
- [➡️ AppSource Publishing](../AppSourcePublishing/) compliance requirements, validation processes, publishing standards, and quality assurance patterns that ensure successful marketplace approval and publication
- [➡️ Core Development](../CoreDevelopment/) the essential building blocks of AL development, from basic language constructs to advanced object implementation patterns.
- [➡️ Integration Deployment](../IntegrationDeployment/) API integrations, accessibility standards, and deployment patterns that ensure robust, compliant AL applications.
- [➡️ Performance Optimization](../PerformanceOptimization/) database optimization, query efficiency, memory management, and AL-specific performance patterns that ensure optimal user experience.
- [➡️ Shared Guidelines](../SharedGuidelines/) consistency, maintainability, and GitHub Copilot effectiveness throughout the entire development process.
- [➡️ Testing Validation](../TestingValidation/) code quality, reliability, and compliance through systematic testing and validation approaches.
